Each operation should have a unique operator (Operation ID)

Noncompliant Code Example (AsyncAPI 2)

asyncapi: '2.4.0'
info:
  title: Unique Operation ID Example API
  version: '1.0.0'
  description: API que demuestra el uso de operationId único en cada operación.
channels:
  /pets:
    description: Canal para operaciones relacionadas con mascotas.    
    subscribe: # Noncompliant {{AAR012: Each operation should have a unique operator (Operation ID)}}
      summary: Operación para recibir la lista de mascotas.
      message:
        contentType: application/json
        payload:
          type: array
          items:
            type: object
            properties:
              id:
                type: string
                description: ID de la mascota.
              name:
                type: string
                description: Nombre de la mascota.
              species:
                type: string
                description: Especie de la mascota.

Compliant Solution (AsyncAPI 2)

asyncapi: '2.4.0'
info:
  title: Unique Operation ID Example API
  version: '1.0.0'
  description: API que demuestra el uso de operationId único en cada operación.
channels:
  /pets:
    description: Canal para operaciones relacionadas con mascotas.    
    subscribe: 
      summary: Operación para recibir la lista de mascotas.
      operationId: hola
      message:
        contentType: application/json
        payload:
          type: array
          items:
            type: object
            properties:
              id:
                type: string
                description: ID de la mascota.
              name:
                type: string
                description: Nombre de la mascota.
              species:
                type: string
                description: Especie de la mascota.
